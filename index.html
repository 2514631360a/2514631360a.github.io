将思考关键词写在草稿纸上，方便思考

手摸 & 画图，猜结论，找性质

手摸：

  • 变量（影响）
  • 不变量
  • 整体变化
  • 局部变化

先考虑暴力（枚举），再优化（考虑调整枚举顺序）考虑简单的东西，避免复杂。

根据条件发散思维，联想相关的东西

分讨

设元：根据题目条件进行设元，可以转化条件，方便考虑贡献，影响等，可以将其当作 DP

  • 序列升序 / 降序，考虑差分换元，转化成差分数组恒大于 / 小于 $0$ 。
  • 区间相关操作，可以考虑前缀和换元，转化成 $s_r$ 与 $s_{l-1}$ 的关系。
  • 异或相关操作，可以考虑异或换元。
  • 特例：不好做时，考虑 DP。

多尝试，不行可以转换思路。

​ 结合部分分。

没有条件，创造条件。

任何问题可以归类于最优化问题：

考虑最优 / 不优条件，影响答案的条件，排除不影响答案的条件

量：不局限于变量 $x$，包括区间，点，边，集合等抽象概念

当题目中出现若干量，考虑量之间的关系（互相推导），尤其是出现明显的等量关系式的时候

按顺序考虑

  • 大小
  • 优先级

函数思想：

将题目 / 条件 / 值 / DP 转化为函数，从函数的角度思考其性质。

如：单调性，单峰 / 单谷，分段，变化规律等。

模型化

时刻 画图/手摸，找规律，猜结论（直观体现特点，将有关的量画图）

  • 先 直观化 ，再 直觉化 。

    • 几何直观（二维平面，数轴，画图，抽象化，类）

    • 代数直观（设状态/元，列式子/表示，考虑如何递推）

  • 观察到性质

如何转化

条件转化：

  • 手摸 & 画图，观察，找性质，特殊到一般，猜结论（猜必要条件，从简单的猜起）

  • 不同类

    • 通过性质，结论，定理，手摸等一系列证明得到

      • 直接加强 / 削弱 / 特殊情况下的条件，一定满足必要 / 充分条件中的一种，证明另一种即可（转化条件的思路）

        考虑若干必要条件，证明充分

    • 修改条件，发现对答案没有影响。（或者排除不优的情况，也就是对答案没有影响的情况）

    • 化简，去除无关量，考虑影响答案的量（转化，化简 DP 状态，最优化（最优/不优），括号匹配去除已匹配的括号）

    • 拆分 / 合并：

      • 把问题拆成几个步骤。考虑已知什么条件/量，怎么做，再考虑怎么得到已知条件（拆分）
      • 多种操作/情况，考虑化简，将其转化/合并。

  • 同类

    • 特殊到一般，考虑状态之间的关系，转化到子问题（DP） 如：$n\rightarrow n+1$，$n\rightarrow n-1$

换元转化

特殊到一般

  • 考虑特殊情况，再调整，推广（一般情况或者其他情况）

特殊情况：一般需要分讨

  • 可以根据题目来定：
    • 大小相关，考虑 $\max \min$ 等。
  • 考虑 整体 / 局部：
    • 图：考虑一条边，一个点，一个点和它的边，环，连通块，点双，边双，割点，割边，强连通分量，dfs 树等
  • 条件相关：
    • 不考虑条件
    • 只考虑某些条件
    • 对条件形式进行变化（加强 / 削弱等）

也要找性质

正做、反做、归纳做。反做：调整顺序，反的顺序，反的递推关系 $A\Rightarrow B$ 变成 $B\Rightarrow A$，枚举 $A$ 变成枚举 $B$ 。

  • $A$ 限制下 $B$ 的最大值，反过来是二分 $B$，求最小的 $A$

  • 给定操作 $A$，求 $B$ 的值，反过来是枚举 $B$，求 $A$ 对 $B$ 的贡献（一般为序列上的操作）

    • 加入 $\Leftrightarrow$ 删除

实际上就是调整顺序

想到一个思路，立刻想是否可以调整顺序。

走不通

  • 找到走不通的点（找性质），调整做法
  • 钦定 / 规范化技巧

对于多种操作/量

  • 将其中若干转化，合并，或者单独考虑。

  • 考虑最优 / 不优条件，影响答案的条件，排除不影响答案的条件。

复杂度过高

  • 找性质缩小范围

  • 考虑暴力是不是对的

  • 考虑最优 / 不优条件，影响答案的条件，排除不影响答案的条件

  • 调整枚举顺序（$2^n$ $\rightarrow$ $\operatorname{dfs}$）

  • 利用重复部分，合并重复部分

构建图论模型

否定/确定一个算法，都要严谨证明其正确性，看能不能举出反例
情况：问题无法表示 $\dots$
